# Constraints — auth (credentials required)

1. Functions exported from src/api/auth.ts MUST verify a credential (bearer token, signed cookie, or API key) before returning a payload that includes `ok: true`.
2. Auth status handlers MUST NOT echo the request body, header keys, or any other untrusted input back to the client; an attacker can use such echoes to confirm what fields the server inspects.
3. Bearer token comparisons MUST use a constant-time string compare (e.g. crypto.timingSafeEqual on equal-length Buffers); standard `===` leaks length and prefix via timing.
4. Every authenticated route MUST resolve a concrete principal (user id or service id) before any database read; routes that proceed with an unresolved principal degrade silently into anonymous reads.
5. Auth modules MUST NOT log or persist the bearer token, signed cookie, or API key in plaintext at any severity level, including DEBUG; secrets in DEBUG logs leak through verbose log shipping pipelines.
